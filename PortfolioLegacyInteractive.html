Legacy Interactive Redesign
Case Study

Business Partner

National Science Teaching Association (NSTA)

Product

An interactive simulation guiding the user–a CDC detective–to figure out which kind of bacteria caused students to get sick.

My Role

UX design.

Outcome

An interactive with updated UI/UX and science content part of the NSTA Science and Food Safety ebook. As a stand-alone product, potentially, it could also sell as a single entity providing another revenue stream.

The Opportunity

From 2013 to 2016, NSTA migrated SciPacks to ePub3s. SciPacks were text-heavy web pages with unintuitive interactives not achieving tangible learning objectives. Through grant funding, NSTA would be able to modernize UI, remove any UX ambiguity, and update the science content, available for sale to anyone, not just our members. This interactive was part of a 200+ revamped collective.

Legacy Interactive

The legacy interactive had the following issues:

  • the user not guided through the screens leading to the user potentially missing critical information to achieve the learning objective;
  • outdated science content;
  • unclear how to achieve the learning objective;
  • too many unnecessary distracting elements;
  • not 508 compliant;
  • no feedback;
  • allowed the user to get stuck in loops leading to frustration; and
  • student-led learning best practices not followed.

Legacy interactive with outdated UI and unituitive UX

Talk to Users

Fortunately, through NSTA conferences across the country, I directly interacted with our potential customers. By having open-ended conversations with teachers, principals, district-heads, etc. I learned a great deal about what they were looking for in ed-tech and what potential obstacles we would have to surmount.

Originally NSTA developed this ebook series for teacher professional and content development. However, in conference conversations, it became abundantly clear many teachers used and wanted to use the interactives with their students either as a class activity or as self-guided learning.

NSTA store at the national conference in St. Louis where I would speak to and observe attendees

Rethinking the Interactive

Knowing these interactives would be used either by middle/high school students or teachers, ensuring the content and UX would function for either demographic was paramount.

This particular interactive is an exploration of information where the user makes inferences to achieve the learning objective. Therefore it is crucial the user review all information without missing screens and keeping track of thoughts without needing access to writing materials. Not only would the UI need to be clear what was active versus inactive, but also what order they should view the screens.

Legacy interactive with no instructions and unclear UX

General UI Design

Overall design goals:

  • remove any unnecessary elements to keep the user focused on the content;
  • avoid gradients on backgrounds increasing accessibility;
  • increasing accessibility by keeping general contrast high; and
  • clear distinction between selected/non-selected and active/non-active elements.

The Suspects screen needed a complete UX and UI redesign. The legacy interactive had all information on one screen. However, there were clear sections where the text could split up.

Legacy suspects screen with a lot of extraneous design elements creating UX ambiguity

Links versus Tabs

The different sections look like navigation links in the initial design round, and the UI failed to differentiate between a viewed screen and a non-viewed screen.

Although a menu-style navigation had made sense, it created a few UX issues. It did not successfully guide the user through the screens, which was the main UX goal, and it did not follow the same design principles as the rest of the interactive navigation in the series style, i.e.link style versus button. In addition to this issue, the change in visual contrast on the link style navigation between viewed/viewing/unviewed was not strong enough, ergo still did not guide the user.

First iteration with menu-style navigation not following previous interactive design rules and creating not enough visual contrast between viewed and non-viewed screens

The second round had a tab-style menu. The tab colors told the user which tab they were currently viewing, had viewed, and still needed to view:

  • gray = viewed
  • blue = viewing
  • green = not viewed

Sequence of tab slection showing how the colors change depending on viewed state

Checkmarks and Colors

The learning objective of the interactive is the user deciding which bacterial species is suspect in the case. The legacy interactive had no method for the user to keep track of their thoughts.

Suspect bacteria information with no way for the user to keep track of their thoughts

Based on conversations with teachers at conferences, we knew both teachers and students did not want to use pen/pencil and paper when reading/interacting with our ebooks. We needed a way for the user to keep track of their prediction and then access it independent of which tab they were on.

We added a title bar with a question mark, checkmark, and cross available at any time. Initially, we colored in these checks: blue for uncertain; green for yes; and red for no. However, the whole internal review team felt the use of color associated with right and wrong.

Suspect bacteria checkmarks using color

However, the user is supposed to make an educated guess only, not take an assessment. With this in mind, we changed all of the options to blue.

Suspect bacteria checkmarks using only blue

Scrollbar

In the first development iteration, the text went off-screen, and it was not obvious you needed to scroll to see the rest. There were a few instances the sentence was incomplete and therefore gave a queue they would need to scroll. However, a few screens had a sentence end at the bottom right of the screen, giving no indication there was more text.

Although there is more text it is not obvious since the scrollbar is not visible

In the second iteration, we added a scrollbar. However–due to a misunderstanding–the scrollbar only appeared when the user tried to scroll. Again not fixing the issue.

On the third round, the scrollbar was visible at all times.

Scrollbar visible at all times

Exploration versus Guided Exploration

The internal team found the interactive to be too exploratory. We knew middle or high schoolers without adult guidance might use the interactive. So we added some structure to the exploration, guiding the user through the interactive, allowing them to move backward and forwards as they wished without potentially missing screens with information. For example, after the first iteration, "Previous" buttons were added, avoiding the user navigating back to the main menu every time to review information.

Labeled buttons guiding the user through the interactive

Final Product

The interactive was exploratory but not to the level where the user felt lost or confused.

We added enough UX to guide the user through the information but still giving them the flexibility to review screens whenever they wanted to.
