Legacy Interactive Redesign
Case Study

Business Partner

National Science Teaching Association (NSTA)

Product

CSI: Food Safety, an interactive simulation guiding the user–a CDC detective–to figure out which kind of bacteria caused students to get sick.

My Role

UX design.

Outcome

An interactive with updated UI & UX and science content part of the NSTA Science and Food Safety ebook. As a stand-alone product, potentially, it could also sell as a single entity providing another value proposition and revenue stream.

The Opportunity

From 2013 to 2016, NSTA migrated SciPacks to ePub3s. SciPacks were text-heavy web pages with unintuitive interactives not achieving tangible learning objectives only available to NSTA members. With grant funding, NSTA would modernize UI, remove UX ambiguity, and update the science content, and removing member barriers of sale.

CSI: Food Safety was part of a 200+ revamped collective of interactives.

Legacy Interactive

The legacy interactive has the following issues:

  • the user is not guided through the screens leading to the user potentially missing critical information to achieve the learning objective;
  • outdated science content;
  • unclear how to achieve the learning objective;
  • too many unnecessary and distracting elements;
  • not 508 compliant;
  • no feedback;
  • allows the user to get stuck in loops leading to frustration; and
  • student-led learning best practices not followed.

Legacy interactive with outdated UI and unituitive UX

Talk to Users

Fortunately, through NSTA conferences across the country, I did user and market research directly interacting with our buyers and decision makers.

Capturing qualitative data from teachers, principals, district-heads, etc. I validated the products’ requirements, customers’ needs in ed-tech, and identified the potential pitfalls discovered in my user research.

Originally NSTA developed this ebook series as professional and content development for teachers. However, in conference conversations, it became abundantly clear many teachers used and wanted to use the interactives with their students either as a class activity or as self-guided learning.

NSTA store at the national conference in St. Louis where I would speak to and observe attendees

Rethinking the Interactive

Knowing these interactives would be used either by middle/high school students or teachers, ensuring the content and UX would function for either demographic was paramount.

This particular interactive is an exploration of information where the user makes inferences to achieve the learning objective. Therefore it is crucial the user review all information without missing screens and keeping track of thoughts without needing access to writing materials. Not only did the UI need to be crystal clear for what was active versus inactive, but also what order they view information.

Legacy interactive with no instructions and unclear UX

General UI Design

Design goals developed through storyboarding and designing user flow are laid out below:

  • remove any unnecessary elements to keep the user focused on the content;
  • increasing accessibility by keeping general contrast high and avoiding background gradients; and
  • distinguishing between selected/non-selected and active/non-active elements.

The Suspects screen needed a complete UX and UI redesign. The legacy interactive had all information on one screen. However, there were clear sections where the text could split up.

Legacy suspects screen with a lot of extraneous design elements creating UX ambiguity

Links versus Tabs

The different sections were styled as navigation links in the initial design round, and the UI failed to differentiate between a viewed screen and a non-viewed screen.

Although a menu-style navigation had made sense, it created a few UX issues. It did not successfully guide the user through the screens, which was the main UX goal, and it did not follow the same design principles as the other interactives' navigation style in the rest of the series, i.e. link versus button. In addition to this issue, the change in visual contrast on the links between viewed/viewing/unviewed was not strong enough, ergo still did not guide the user.

First iteration with menu-style navigation not following previous interactive design rules and not creating enough visual contrast between viewed and non-viewed screens

The second round had a tab-style menu. The tab colors told the user which tab they were currently viewing, had viewed, and still needed to view:

  • gray = viewed
  • blue = viewing
  • green = not viewed

Sequence of tab slection showing how the colors change depending on viewed state

Checkmarks and Colors

The learning objective of the interactive is the user deciding which bacterial species is suspect in the case. The legacy interactive had no method for the user to keep track of their thoughts.

Suspect bacteria information with no way for the user to keep track of their thoughts

Based on qualitative research at conferences, we knew both teachers and students did not want to use pen/pencil and paper when reading/interacting with our ebooks. We needed a way for the user to keep track of their prediction and then access it independent of which tab they were on.

We added a title bar with a question mark, checkmark, and cross available at any time. Initially, we colored in these checks: blue for uncertain; green for yes; and red for no. However, the whole internal review team through A/B testing concluded the use of color created an association between correct and incorrect.

Before: Suspect bacteria checkmarks using color signifying right and wrong

However, the user is supposed to make an educated guess, not take an assessment. With this in mind, all options changed to blue.

After: Suspect bacteria checkmarks using only blue

Scrollbar

In the first development iteration, the text went off-screen, and it was not obvious you needed to scroll to see the rest. There were a few instances the sentence was incomplete and therefore gave a queue they would need to scroll. However, a few screens had a sentence end at the bottom right of the screen, giving no indication there was more text.

Although there is more text it is not obvious since the scrollbar is not visible

In the second developement iteration, we added a scrollbar. However the scrollbar only appeared when the user tried to scroll. Again not fixing the issue.

On the third development round, the scrollbar was visible at all times.

Scrollbar visible at all times

Exploration versus Guided Exploration

The internal team through usability testing found the interactive to be too exploratory. Knowing middle or high schoolers without adult guidance might use the interactive, structure to the exploration was added, guiding the user through the interactive, allowing them to move backward and forwards as they wished without potentially missing screens with information. For example, after the first iteration, "Previous" buttons were added, avoiding the user navigating back to the main menu every time to review information.

Labeled buttons guiding the user through the interactive

Final Product

The interactive was exploratory without the potential of getting stuck in loops leading to frustration.

The UX guided the user through the information while giving them flexibility to review at their discretion.
